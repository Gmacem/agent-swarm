# Summarize Docs

Summarize documentation for a library, API, or tool.

## Arguments

- `$ARGUMENTS` — What to summarize (e.g. "Prisma ORM docs", "AWS S3 API", URL to documentation)

## Instructions

1. Find and read the relevant documentation.
2. Extract and summarize:
   - What the tool/library does.
   - Key concepts and terminology.
   - Getting started steps.
   - Important configuration options.
   - Common patterns and best practices.
   - Gotchas and known limitations.
3. Keep the summary concise and actionable — focus on what the team needs to know.
4. Save the summary to `knowledge/research/<topic>.md`.
5. Report the summary and file path.
